Документация по драйверу uartlite в Vitis (бывший SDK)

Отличный и самый правильный вопрос! Работа с любой периферией начинается с изучения документации. Для драйвера uartlite в Vitis есть два основных источника информации.

1. Официальная документация (онлайн)

Самый актуальный и полный источник — это официальная документация AMD (Xilinx), размещённая на GitHub Pages:

  • Обзор драйвера (XUartLite)
    Ссылка: https://xilinx.github.io/embeddedsw.github.io/uartlite/doc/html/api/index.html
    Что там есть: Подробное описание драйвера, его возможностей, режимов работы (опрос, прерывания), инициализации и важные примечания (например, что скорость UART задаётся только аппаратно, а FIFO имеет фиксированный размер в 16 байт).
  • Документация по API (Справочник функций)
    Ссылка: https://xilinx.github.io/embeddedsw.github.io/uartlite/doc/html/api/xuartlite_8h.html
    Что там есть: Это самое главное. Здесь вы найдете алфавитный список и подробное описание всех функций драйвера xuartlite. Для каждой функции указано её назначение, параметры и возвращаемое значение. Это настольная книга при работе с этим драйвером.

2. Разница между функциями (по документации)

Изучив документацию, вы увидите четкое разделение:

Высокоуровневый API (xuartlite.h)

  • Функции: XUartLite_Initialize(), XUartLite_Send(), XUartLite_Recv(), XUartLite_IsSending().
  • Как работают: Эти функции работают с экземпляром структуры XUartLite. Они «умные» — сами управляют FIFO, проверяют его состояние, ждут, когда можно отправить следующий байт и т.д. Функция XUartLite_Send() неблокирующая, но она всегда знает, сколько байт реально было отправлено в FIFO [1]. Для ожидания окончания отправки всех данных используется функция XUartLite_IsSending(), которая возвращает TRUE, пока передатчик занят [2]. Именно эту связку рекомендуется использовать в вашем проекте.

Низкоуровневый API (регистровый уровень, xuartlite_l.h)

  • Функции: XUartLite_SendByte(), XUartLite_RecvByte(), XUartLite_IsTransmitFull(), XUartLite_IsReceiveEmpty().
  • Как работают: Это, по сути, удобные макросы для прямой записи и чтения регистров UART по базовому адресу [3]. Они не используют структуру XUartLite (или используют её только для получения адреса). Они работают очень просто:
    • XUartLite_SendByte() пытается записать байт в регистр передатчика. Но, как вы выяснили из своего опыта, она может не проверять, готов ли FIFO принять этот байт [4]. Именно поэтому ваш код и зависал — функция пыталась записать байт в полный FIFO и, в зависимости от реализации драйвера, либо теряла данные, либо входила в бесконечный цикл ожидания, который не мог завершиться из-за неверных условий.
    • XUartLite_RecvByte() — аналогично, это блокирующая функция, которая будет ждать, пока в приемном FIFO не появятся данные [5].

3. Где найти примеры кода

Документация — это хорошо, но часто проще учиться на готовых примерах.

  • Встроенные примеры Vitis (SDK): Самый лучший способ — обратиться к примерам, которые идут вместе с драйвером. Они находятся в директории установки Vitis. Для вашего uartlite путь будет примерно таким: Vitis/2023.2/data/embeddedsw/XilinxProcessorIPLib/drivers/uartlite_vX_Y/examples/ (где X_Y — версия драйвера) [6]. Там вы найдете xuartlite_low_level_example.c (пример использования низкоуровневых функций) и xuartlite_intr_example.c (пример с прерываниями) [7].
  • Форумы сообщества (AMD Support): На форуме adaptivesupport.amd.com можно найти множество обсуждений и фрагментов кода. Например, там есть примеры использования XUartLite_SendByte для простейшей отправки [8] и обсуждение проблем с блокировкой при приеме [9].
