Куда можно сохранить команды для терминала Ubuntu

В Ubuntu (и других Linux-системах) есть несколько удобных способов сохранить команды для терминала, чтобы их можно было быстро использовать в будущем. Вот основные варианты:

1. Файл с алиасами (~/.bashrc или ~/.bash_aliases)

Можно добавить алиасы (короткие команды) в файл конфигурации оболочки.

  1. Откройте файл .bashrc или .bash_aliases в редакторе: 
    nano ~/.bash_aliases

    (Если файла нет, он создастся автоматически.)

  2. Добавьте команды в формате: 
    alias короткая_команда='длинная_команда'
    Например: 
    alias update='sudo apt update && sudo apt upgrade -y'
alias ll='ls -la'
  3. Сохраните (Ctrl+O, Enter) и закройте (Ctrl+X).
  4. Примените изменения: 
    source ~/.bash_aliases
    Теперь можно использовать update вместо полной команды.

2. История команд (~/.bash_history)

Все введённые команды сохраняются в файле:

cat ~/.bash_history

Но это неудобно для поиска, лучше использовать:

history | grep "часть_команды"

3. Отдельный текстовый файл

Можно создать файл вручную, например:

mkdir -p ~/Documents/commands
nano ~/Documents/commands/my_commands.txt

И записывать туда полезные команды.

4. Системные файлы (/usr/local/bin/)

Если команда используется часто, можно сделать её исполняемым скриптом:

  1. Создайте файл: 
    sudo nano /usr/local/bin/myscript
  2. Добавьте #!/bin/bash и команды: 
    #!/bin/bash
echo "Hello, this is my script!"
  3. Дайте права на выполнение: 
    sudo chmod +x /usr/local/bin/myscript
    Теперь можно вызывать просто myscript.

5. Менеджеры заметок (Obsidian, Joplin, Notion)

Если команды нужно хранить с пояснениями, удобно использовать:

  • Obsidian (локальные Markdown-файлы)
  • Joplin (синхронизация между устройствами)
  • Notion (облачное хранение)

6. Инструменты для хранения команд (tldr, cheat)

  • tldr – упрощённые man-страницы с примерами: 
    sudo apt install tldr
tldr tar
  • cheat – позволяет сохранять свои шпаргалки: 
    sudo apt install cheat
cheat tar

Вывод

  • Для быстрого доступа → алиасы в ~/.bash_aliases.
  • Для хранения с пояснениями → текстовый файл или Obsidian/Notion.
  • Для сложных скриптов → /usr/local/bin/.
  • Для истории → history или ~/.bash_history.
