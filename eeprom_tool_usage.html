eeprom_tool_fin.c

📌 Описание программы

Программа eeprom_tool_fin.c предназначена для побайтовой записи и чтения данных из EEPROM через UART (RS-232).

Протокол обмена:

  • Вход в режим программирования: отправка байта 'P' (0x50), ожидание ответа "PROG" (4 байта).
  • Запись байта: команда 'W' + адрес (2 байта, старший → младший) + байт данных, ожидание ответа "OK".
  • Чтение байта: команда 'R' + адрес (2 байта), устройство возвращает один байт данных (без дополнительного подтверждения).

Настройки UART по умолчанию: 115200 8E1 (115200 бод, 8 бит данных, чётность Even, 1 стоп-бит, без управления потоком).

🛠 1. Компиляция

Убедитесь, что установлен компилятор GCC:

sudo apt update
sudo apt install build-essential

Скомпилируйте программу:

gcc -o eeprom_tool eeprom_tool_fin.c

Для компиляции с отладочной информацией:

gcc -g -o eeprom_tool eeprom_tool_fin.c

🔌 2. Подготовка UART-порта

Программа по умолчанию использует порт /dev/ttyUSB0. Если ваш адаптер создаёт другой порт (например, /dev/ttyACM0), измените макрос в коде:

#define UART_DEVICE "/dev/ttyUSB0"

Проверьте доступные порты:

ls -l /dev/ttyUSB*

Добавьте пользователя в группу dialout для доступа к порту:

sudo usermod -a -G dialout $USER

После этого выйдите из системы и зайдите заново (или перезапустите оболочку).

✍️ 3. Режим записи (по умолчанию)

Входной файл

Создайте файл repacked_4bytes.txt в той же папке, что и исполняемый файл. Каждая строка должна содержать ровно 4 шестнадцатеричных числа, разделённых пробелами или табуляцией.

Пример содержимого:

13 52 9A CD
FE 01 02 03
A0 B1 C2 D3

Пустые строки игнорируются.

Запуск

./eeprom_tool

Программа выполнит:

  • открытие UART,
  • вход в режим программирования,
  • чтение файла построчно,
  • запись 4 байт последовательно, начиная с адреса 0x06 (задаётся макросом START_ADDR),
  • ожидание OK после каждого байта,
  • вывод статистики.
Пример вывода в режиме записи:

📖 4. Режим чтения

Запуск с ключом -r:

./eeprom_tool -r [start_addr] [count]
  • start_addr — начальный адрес (шестнадцатеричный или десятичный), по умолчанию 0x05.
  • count — количество байт для чтения, по умолчанию 400.

Примеры:

./eeprom_tool -r                   # читает 400 байт с адреса 0x05
./eeprom_tool -r 0x10 32            # читает 32 байта с адреса 0x10
./eeprom_tool -r 0 8               # читает 8 байт с адреса 0x00
Пример вывода в режиме чтения: 
=== Режим ЧТЕНИЯ ===
Адрес начала: 0x0005, количество байт: 16
Чтение...

0x0005: 0x2E
0x0006: 0x13
0x0007: 0x52
0x0008: 0x9A
...
Прочитано байт: 16

⚙️ 5. Настройка параметров в коде

Основные макросы в начале файла:

#define UART_DEVICE     "/dev/ttyUSB0"   // UART-порт
#define BAUD_RATE       B115200          // скорость
#define START_ADDR      0x06             // стартовый адрес для записи
#define READ_START      0x05             // стартовый адрес для чтения (по умолчанию)

Для изменения задержек отредактируйте функции:

  • В write_byte после получения OK стоит usleep(100000); (100 мс).
  • В read_byte после отправки команды стоит usleep(50000); (50 мс) и после получения байта usleep(100000); (100 мс).

Таймауты: expect_ok(1000) (1 сек), recv_byte(..., 2000) (2 сек).

❗ 6. Частые проблемы и решения

ПроблемаРешение
Ошибка открытия UART: No such file or directory Проверьте наличие адаптера: ls /dev/ttyUSB* или dmesg | grep tty. Убедитесь, что порт существует.
Permission denied при открытии порта Добавьте пользователя в группу dialout и перелогиньтесь. Или запустите с sudo.
Таймаут ожидания PROG Проверьте питание EEPROM, правильность подключения, совпадение скорости и чётности.
Таймаут ожидания O при записи Увеличьте задержку после отправки данных в write_byte (например, до 200 мс).
Чтение обрывается после первого байта Увеличьте задержку после получения байта в read_byte (например, usleep(200000)).
Ошибка парсинга строки в файле Убедитесь, что числа разделены пробелами и нет лишних символов (запятых, точек).
💡 Совет: Если ваше устройство работает без чётности, закомментируйте строки с PARENB и PARODD в функции uart_init и добавьте tty.c_cflag &= ~PARENB;.

📎 7. Дополнительные замечания

  • Программа использует чётность Even. Если устройство требует другого режима, измените настройки в uart_init.
  • Для медленных EEPROM увеличивайте все задержки в 2–5 раз.
  • Режим чтения по умолчанию читает 400 байт с адреса 0x05 — при необходимости измените макрос READ_START.
  • Все временные задержки указаны в микросекундах (usleep).
